<p>As of August 31, 2022, the Message Content Intent will have become a privileged intent and verified bots that are in 75 or more servers will need to either be approved for the intent or have migrated to slash commands. You can read more about this <a href="https://support-dev.discord.com/hc/en-us/articles/4404772028055-Message-Content-Privileged-Intent-FAQ">here</a>.</p>
<p>We are introducing a more visible way to educate users on a bot's available features. We'll be testing commands badges on bot profiles to help draw attention to the fact that a bot supports commands.</p>
<p><strong>If your app has at least one global command, you will see a shiny new badge on its profile!</strong></p>
<p>Here's some important info:</p>
<ul>
<li>The badge will be displayed on <strong>the bot's user profile.</strong>
</li>
<li>When hovering over the badge, the tool tip will state: <strong>"Supports Commands".</strong>
</li>
<li>Clicking the badge will <strong>redirect users to <a href="https://discord.com/blog/welcome-to-the-new-era-of-discord-apps">a blog post</a></strong> where they can learn about slash commands, what additional features bots can support, and other FAQs.</li>
<li>All bots and apps are eligible! <strong>Slash commands, user commands, and message commands are all applicable.</strong>
<ul>
<li>All you need to do is <strong>register at least one global command</strong> for your app.</li>
<li>Learn more about global scoped application commands <a href="https://discord.com/developers/docs/tutorials/upgrading-to-application-commands">here.</a>
</li>
</ul>
</li>
<li>If your app already meets this criteria, <strong>there's nothing you need to do</strong>.</li>
<li>Here's an example of what the badge looks like on a bot's profile:</li>
</ul>
